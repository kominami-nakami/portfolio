# ポートフォリオ仕様書（Cloudflare Pages）

## 1. 目的
- Cloudflare Pages で公開できる静的ポートフォリオサイトを作成する
- 独自サブドメイン `portfolio.kominami-nakami.com` でアクセス可能にする

## 2. 構成
- `index.html`：ページ本体
- `styles.css`：見た目（レスポンシブ対応）

## 3. ページ要件
- ヒーロー（名前・肩書き・自己紹介）
- 制作物カード（3件）
- スキル一覧
- 連絡先（メール・GitHub）
- モバイル/PC の両方で崩れないこと

## 4. ローカル確認
このフォルダで `index.html` をブラウザで開いて表示確認する。

## 5. Cloudflare Pages 公開手順
### 5.1 GitHub リポジトリ作成
1. `ポートフォリオ` フォルダを GitHub リポジトリへ push

### 5.2 Pages プロジェクト作成
1. Cloudflare Dashboard を開く
2. **Workers & Pages** → **Create application** → **Pages** → **Connect to Git**
3. 対象リポジトリを選択
4. Build settings は以下
	- Framework preset: `None`
	- Build command: （空欄）
	- Build output directory: `/`
5. **Save and Deploy**

## 6. 独自サブドメイン設定（`portfolio.kominami-nakami.com`）
### 6.1 Pages 側
1. Pages プロジェクト → **Custom domains** → **Set up a custom domain**
2. `portfolio.kominami-nakami.com` を入力して追加

### 6.2 DNS 側（同じ Cloudflare ゾーン管理の場合）
- 通常は自動で DNS レコードが作成される
- 自動作成されない場合は、Cloudflare が表示する対象へ `CNAME` を作成

### 6.3 DNS 側（ドメインを他社DNSで管理している場合）
- DNS レコードを以下で作成
	- Type: `CNAME`
	- Name: `portfolio`
	- Target: Pages 追加時に Cloudflare が提示する `*.pages.dev` の値
- DNS 反映後、Cloudflare Pages 側でドメイン検証が完了する

### 6.4 SSL/TLS
- Cloudflare の Universal SSL が有効なら自動で HTTPS 化される

## 7. 運用メモ
- `main` ブランチへ push すると自動デプロイ
- 表示更新が遅い場合は Pages の最新デプロイ完了を確認
- 独自ドメイン接続直後は DNS 反映に時間がかかることがある
